##COST ITEMS
###Allowance
 
An allowance is a bucket of money included in the contract to cover the cost of certain items that have not yet been selected or specified. Typically, these are the items the contractor knows will need to be selected at some point but the choice doesn’t impact the building process. For example, a home builder may include an allowance for the appliance package. The choice doesn’t change how they would complete the install but the price can vary greatly from low end to high end. The allowance acts as a placeholder bucket of money to create a semi accurate project estimate which also helps the homeowner justify the total loan amount with their bank.
###Allowance Type
 
Determine how you would like to handle your profits by choosing one of the three allowance types. Price allowance includes profit margin in totals. Cost Allowance maintains fixed profit basecd on the initial allowance amount. Cost & Fee Allowance collects an additional profit on overages, if the customer selection exceeds the allowance amount.
###Cost Code: ID
 
Each cost code in JobTread has an ID that uniquely identifies it. This ID ensures accuracy when referencing cost codes across budgets, reports, and integrations. It is automatically assigned by the system.
###Cost Code: Name
 
The name of a cost code provides clarity on what the code represents. Common examples include 'Framing' or 'Plumbing.' Names make cost codes easier to use when estimating, budgeting, and reporting. You will set these up in your Job Costing tab, within your organization settings.
###Cost Code: Number
 
Cost code numbers organize cost categories in a standardized way. For example, '0300' might represent concrete. Using numbers ensures consistency across jobs and aligns with construction accounting best practices.
###Cost Code: Parent Cost Code
 
Parent cost codes group individual cost codes into broader categories. For instance, '0300 Concrete' may contain subcodes for footings, slabs, and walls. This hierarchy helps organize budgets and improves reporting clarity.
###Cost Code: Parent Cost Code: ID
 
Every parent cost code has a system assigned ID. This ensures that parent child cost code relationships are tracked accurately in JobTread. It allows for structured reporting and integration consistency.
###Cost Code: Parent Cost Code: Name
 
The parent cost code name describes the broader category that contains subcodes. For example, 'Concrete' may be the parent name for codes like footings or slabs. It provides an organized structure for budgeting and reporting.
###Cost Code: Parent Cost Code: Number
 
The number assigned to a parent cost code represents the category in a structured system. For example, '0300' could be concrete, with child codes like '0310' for slabs representing a subcategory. This keeps cost tracking consistent and clear.
###Cost Item
 
Line item costs associated with a job. This includes labor, materials, subcontractor costs, equipment rental, permitting, and more. Cost items can be added to the catalog to quickly copy them to future jobs.
###Cost Item ID
 
Every cost item in JobTread has a unique ID. This ID ensures cost items are properly tracked across budgets, invoices, and reports. It prevents errors and maintains consistency across records.
###Cost Type: Created At
 
This is the system timestamp for when a cost type record was created. Tracking creation dates provides an audit trail for setup and ensures visibility into when financial structures were established.
###Cost Type: ID
 
Each cost type in JobTread has an ID that uniquely identifies it. This ID ensures that cost types are referenced accurately in budgets, invoices, and reports. It is automatically assigned by the system.
###Cost Type: Is Taxable
 
This field shows whether a cost type should have tax applied. Taxable cost types affect how invoices and budgets are calculated. Setting this correctly ensures compliance with tax reporting requirements.
###Cost Type: Margin
 
The margin for a cost type represents the expected profit percentage applied to that category of costs. It helps standardize pricing and ensures consistent profitability across similar types of expenses in JobTread.
###Cost Type: Name
 
The name identifies and labels the cost type and provides clarity in estimates, budgets, and invoices. Examples might include 'Labor' or 'Materials.' Clear naming conventions improve team communication and financial tracking.
###Created At
 
Every record in JobTread has a creation timestamp. This helps track when entries were added and provides an audit trail. It ensures data accuracy and supports project management oversight.
###Files Count
 
Numeric indicator of how many files are linked to a record in JobTread. Files may include contracts, photos, or invoices. Tracking file count helps ensure proper documentation is stored for each job or customer.
###Is Taxable
 
This flag indicates whether tax should be applied to a specific cost item or line. It affects how budgets and invoices are calculated. Setting this properly ensures compliance with tax laws.
###Margin
 
Margin represents the percentage of profit applied on top of job costs. It ensures you are making money beyond just covering expenses. JobTread uses margin in budgets and reports to calculate profitability.
###Markup
 
Markup is the percentage added to costs to determine the customer price. Markup is calclated by subtracting the cost of an item from it's price, then diving it by the price and then multiplying that number by 100. Unlike margin, markup is calculated from the cost base rather than profit. JobTread allows you to track both markup and margin for accurate pricing.
###Quantity
 
Quantity represents how many units of a cost item are required for a job. For example, 500 square feet of flooring or 20 hours of labor. Setting quantities accurately ensures realistic budgets and invoices.
###Quantity Formula
 
A quantity formula automates the calculation of necessary units based on job parameters. For example, it could calculate paint gallons needed using wall square footage. This improves estimating accuracy and saves time.
###Require Specification Approval
 
This setting requires customer approval of a cost item in order for it to be included as specification item as it becomes active in a job. This helps contractors manage change orders and ensures that selections are authorized before work begins.
###Specification
 
Specifications define materials, products, or finishes to be used in a job. For example, “12x24 porcelain tile in gray finish.” JobTread tracks specifications in the specifications tab to keep teams aligned and customers informed.
###Unit
 
Units define how a cost item is measured, such as square feet, hours, or each. Proper units ensure quantities and costs are consistent across jobs. They provide clarity in estimating, purchasing, and billing.
###Unit Cost
 
Unit Cost represents the contractor’s expense for one unit of a cost item, such as $3.00 per square foot of flooring. Accurate unit costs ensure budgets reflect real expenses and allow for precise profitability tracking.
###Unit Cost Formula
 
A unit cost formula automates the calculation of a cost per unit based on job data. For example, labor cost per square foot could be derived from crew size and hourly rate. This streamlines estimating and reporting.
###Unit Price
 
Unit Price is the billed rate to the customer for one unit of a cost item, such as $6.00 per square foot of flooring. This includes cost plus markup or margin. Tracking unit price ensures invoices align with estimates.
###Unit Price Formula
 
A unit price formula automates how the customer’s price is calculated from cost data. It may include markup, labor rates, or material factors. This ensures accurate pricing and consistency across jobs.
###Write In
 
Items or materials that the Customer has personally chosen and specified, typically outside of the standard or pre approved options. Within JobTread, allowing for Customer write ins will enable your Customer to update the description, quantity and files for this item when enabled on a Customer Order

##DOCUMENTS
###Add Files
 
This option allows contractors to add supporting documents, such as terms and conditions or brochures. Files are attached automatically whenever the document type is created.
###Allow Partial Payments
 
When enabled, customers can make partial payments on invoices using credit card or ACH. If disabled, invoices must be paid in full. This provides flexibility in managing customer payments.
###Balance
 
Balance shows how much money is still unpaid on an invoice or contract. It helps track outstanding receivables. Monitoring balances ensures cash flow is managed effectively.
###Bid Request
 
A bid request is a document sent to vendors to collect pricing for materials or services. Contractors use them to compare quotes and choose the best option for the project. JobTread bid requests streamline vendor communication and documentation.
###Change order
 
A change order is used to record an amendment to the original contract, typically involving changes in the scope of work, materials, or costs. Change orders are used to document changes, collect customer signatures, and make adjustments to the Budgeted Cost column in a job budget.
###Cost
 
Cost reflects the actual expense for materials, labor, or services recorded in JobTread. It is used to calculate profitability. Accurate cost tracking is essential for financial reporting.
###Cost With Tax
 
This field shows cost amounts after applicable taxes are added. It ensures estimates and invoices reflect full amounts customers must pay. Accurate tax application is critical for compliance.
###Customer Invoice
 
Customer invoices are sent to clients to request payment for labor, materials, or milestones. They include payment terms and can allow for partial or full payments. In JobTread, invoices are tied directly to job budgets and financial tracking.
###Customer Order
 
A customer order creates an official record of what the client has approved. It captures customer selections, scope of work, and pricing. This document is typically signed by the customer and becomes part of the project contract.
###Description
 
The Description field allows contractors to include project details, scope of work, or job notes. It appears at the top of the document for the recipient. Using descriptions ensures the customer understands the context of the document.

###Document ID
 
Every document in JobTread, such as an invoice or contract, has a system assigned ID. This ID ensures accuracy in tracking, reporting, and integrations.
###Document Name
 
Document Name is the heading shown to customers or vendors when they receive a document. It should clearly identify the type of document and job it relates to. A professional name improves clarity and customer communication.
###Document Template
 
Document templates are pre configured formats for common documents like bids, invoices, and purchase orders. JobTread provides default templates that can be customized or duplicated. Templates save time, ensure consistency, and standardize communication with customers and vendors.
###Due Days
 
Due Days defines the timeframe a customer or vendor has to meet the document’s terms, such as payment or delivery. Setting clear due days ensures expectations are communicated.
###Email Message
 
Message that accompanies a document that can templated or be customized before sending. Templating this message can save time and ensures consistent, professional communication with clients and vendors.
###External ID
 
External IDs are used when JobTread data links to third party systems like QuickBooks. They ensure records match correctly between platforms. This prevents duplication and maintains data integrity.
###Footer
 
The footer provides space for contractual language, disclaimers, or payment terms. It appears at the bottom of the document. Using a footer ensures important legal or financial details are always included.
###Payment Schedule Tab
 
The Payment Schedule Tab allows contractors to define installment payments within customer orders using invoices, progress invoices and deposits. This is useful for jobs with consistent billing structures. It helps standardize and automate invoicing.
###Prepared By
 
Prepared By auto populates with the JobTread user’s contact information when a document is created. This shows who created the document and provides a point of contact for recipients. This can also be templated within each Document Template to show a consistent "prepared by" sender.
###Price
 
Price is the customer facing amount set for a job, item, or service. It is typically calculated by adding markup or margin to the cost. Tracking price ensures estimates and invoices reflect agreed amounts.
###Price With Tax
 
This field shows the price after applicable sales tax is added. It ensures customers see the full amount due. Including tax in pricing helps avoid confusion during invoicing.
###Purchase Order
 
Purchase Orders, aka POs, are a formal document issued from a buyer to a seller, specifying the products, quantities, and agreed prices for the products and services. For contractors, POs are used to formalize purchases with vendors but also act as contracts with subcontractors to perform work (often called a work order). POs allow you to commit funds without paying out the funds yet.

Actual Cost is the true expense recorded for materials, labor, or services. It is compared against budgeted costs to measure financial performance. Tracking actual costs ensures profitability analysis is accurate.
###Backcharge
 
A backcharge is a request for reimbursement sent to a contractor after deficiencies were found in the work they performed, if they didn’t complete a portion of the work, or if they caused damage while on site.

###Contingency
 
Contingencies are a set dollar amount included on the contract meant for unforeseen costs. Instead of billing the client for small adjustments, the contractor would pull from the contingency fund first. This can help move the job along at a quicker pace because the money has already been approved to spend.
###Cost codes
 
Cost codes are the accounting buckets that all cost items will fall under. Every cost item in the system is going to be tied to a cost code. Because cost codes are what connect to QuickBooks Products/Services, they shouldn’t change often. The idea is you may add thousands of cost items over time but they should all fall under an existing code.
###Cost groups
 
Cost groups (aka cost templates) are groups of cost items. Groups can be created in the catalog as templates to make estimating quicker. They also allow you to organize the budget into smaller chunks to make large budgets more digestible. Cost groups can include subgroups, up to 5 levels deep.
